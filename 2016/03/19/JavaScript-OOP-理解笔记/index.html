      JavaScript-OOP相关-笔记

      Post on 2016-03-19

      对象的创建方法

      第一种是对象字面量的形式

      person = {......}

      第二种是构造函数的方式

      function Person(name, age){ 

//当调用new Person的时候，在执行第一行代码前，先生成一个Person对象，并将对象在内存中的 
//索引赋值给this关键字，此时可以通过this关键字操作新生成的对象，如下面的添加属性或方法 

this.name = name; //this关键字不能少。为当前对象，即this关键字引用的对象的name属性赋值 
//，实际相当于为当前对象添加name属性后,再为其name属性赋值。 
this.age = age; 

this.showName = function(){ 
alert(this.name); 
} 
this.showAge = function(){ 
alert(this.age); 
} 
} 

var obj1 = new Person("Koji", 22); //生成一个Person对象 
var obj2 = new Person("Luo", 21);

      这种方式的缺点是，每次new的时候都会创造一个新的方法，浪费内存。

      第三种方法是原型方法

      function Person(){} //定义一个空构造函数，且不能传递参数 

//将所有的属性的方法都赋予prototype属性 

Person.prototype.name = "Koji"; //添加属性 
Person.prototype.age = 22; 

Person.prototype.showName = function(){ //添加方法 
alert(this.name); 
} 

Person.prototype.showAge = function(){ 
alert(this.age); 
} 

var obj1 = new Person(); //生成一个Person对象 
var obj2 = new Person();

      缺点是上面的代码通过obj1向obj1的属性array添加元素的时候，obj2的arra属性的元素也跟着受到影响，原因就在于obj1和obj2对象的array属性引用的是同一个Array对象，那么改变这个Array，对象，另一引用该Array对象的属性自然也会受到影响

      第四种是混合方法，混合了原型和构造函数方法 

      function Person(name, age) { 
this.name = name; 
this.age = age; 
this.array = new Array("Koji", "Luo"); 
} 

Person.prototype.showName = function() { 
alert(this.name); 
} 

Person.prototype.showArray = function() { 
alert(this.array); 
} 

var obj1 = new Person("Koji", 22); //生成一个Person对象 
var obj2 = new Person("Luo", 21); 

obj1.array.push("Kyo"); //向obj1的array属性添加一个元素 

obj1.showArray(); //Koji,Luo,Kyo 
obj1.showName(); //Koji 
obj2.showArray(); //Koji,Luo 
obj2.showName(); //Luo

      这样，不仅不同实例之间属性是私有的，方法是共享的
      这种方法是最常用的

      还有一种方法是使用Object.create() 新特性，优点是直观，能直接创造一个具有指定原型的对象，缺点是有一些浏览器不支持

      apply和call

      改变this的上下文用。也用于对象的（属性）继承。
      将括号内部的对象的this指向外部的那个对象。

      判断数据类型的几种形式

      1. toString.call 准确判断数据类型：Array、String、Object、RegExp、Number、Undefined、Null、Date、Boolean、Function
      2. typeof 粗略判断数据类型：undefined、number、string、boolean、object、function
      3. instanceOf 判断一个函数是否为另一个函数的实例。

      继承

      1. 对象冒充的方法继承，一般用call/apply 来继承对象的属性 不继承方法是因为每继承一次会在内存中新建一个方法
      2. 原型链继承，用来继承对象的方法和属性。缺点是实例化子类时不能将参数传给父类
      3. 混合继承 成员变量采用对象冒充方式，成员方法采用原型链方式
      4. object.create（） 新标准 唯一缺点是兼容性

      变量提升和函数提升

      js中会默认把定义变量这个行为提升到函数顶部
      如果是下面这一段代码 会返回 undefined

      var v='Hello World'; 
(function(){ 
alert(v); 
var v='I love you'; 
})()

      原因是程序将var v提升到了alert（v）之上
      这也是js书写规范中建议将对变量的定义提到顶部的原因，防止出错。

      另外，js也有函数提升这一特性，但是只限函数声明，函数表达式是没有用的
      如下，第一段代码成功执行，而第二段不行

      function myTest(){ 
foo(); 
function foo(){ 
alert("我来自 foo"); 
} 
} 
myTest();

      function myTest(){ 
foo(); 
var foo =function foo(){ 
alert("我来自 foo"); 
} 
} 
myTest();
